# Changelog

## 2026-06-14

- Добавлена обратная webhook-проверка устройств: каждое устройство получает персональный URL с секретным token.
- Добавлен schema v2 для `config.json`: `ping_hosts`, `webhook_devices`, `_schema_version`, `webhook_base_url`, `trust_proxy_headers`.
- Добавлена автоматическая миграция старого плоского `config.json` в schema v2 при первом запуске.
- Запись конфига переведена на атомарный `os.replace()` и общий `threading.RLock`.
- Добавлен webhook endpoint `GET|POST /webhook/<token>` для heartbeat.
- Добавлен webhook worker, который переводит устройство в `offline`, если heartbeat не приходит дольше `interval_seconds * missed_heartbeats`.
- Добавлены WebUI-формы и таблица для webhook-устройств: название, место, тип, интервал, пропуски до offline, статус, последний heartbeat и last IP.
- Добавлен `Webhook base URL` в настройки WebUI.
- Добавлена генерация инструкций и готовых команд для Linux, macOS, Windows и RouterOS.
- Добавлены экспериментальные инструкции для Keenetic и UniFi.
- Добавлена смена webhook token с переводом устройства обратно в `pending`.
- Расширены автотесты: миграция конфига, webhook routes, online/offline transitions, инструкции, Windows-валидация и регрессия ping-проверок.
- Обновлена русская и английская документация.

## 2026-06-09

- Добавлены автотесты на `pytest` и GitHub Actions workflow для запуска тестов на push/PR.
- Исправлено чтение `.env`: путь к env-файлу теперь вычисляется при вызове `load_env_file()`, а не фиксируется при загрузке модуля.
- Добавлен двуязычный WebUI на English / Русский с английским языком по умолчанию.
- Выбранный язык интерфейса сохраняется в `_settings.language`.
- Из подсказок WebUI и дефолтного конфига убраны личные IP/тексты; по умолчанию используются `192.168.0.1`, `Host is available`, `Host access lost`.
- В WebUI добавлено редактирование существующих хостов.
- Первая проверка нового IP теперь отправляет Telegram-сообщение, даже если прошлое состояние было `unknown`.
- Ручная проверка IP теперь всегда отправляет Telegram-сообщение с префиксом `Ручная проверка: `.
- В WebUI добавлена ручная проверка каждого IP.
- В WebUI добавлена ручная проверка SOCKS5-прокси через Telegram API.
- Общая логика проверки хоста теперь используется и фоновым worker-ом, и ручной кнопкой проверки.
- Telegram-настройки `TOKEN` и `CHAT_ID` вынесены из `app.py` в файл `/root/ping_manager/.env`.
- Добавлен пример файла переменных `ping_manager/.env.example`.
- Unit-файл systemd подключает `/root/ping_manager/.env` через `EnvironmentFile`.
- Исправлена отправка Telegram-сообщений после добавления SOCKS5-прокси.
- Добавлена валидация настроек прокси: пустой IP, некорректный порт и порт вне диапазона больше не ломают отправку.
- Для SOCKS5 используется схема `socks5h://`, чтобы DNS-запросы Telegram также проходили через прокси.
- Добавлен fallback на прямую отправку, если прокси недоступен или в системе не установлен пакет поддержки SOCKS (`python3-socks` / `PySocks`).
- Добавлена проверка HTTP-ошибок и ответа Telegram API, чтобы ошибки отправки больше не игнорировались молча.
- Убран `parse_mode=Markdown` из Telegram-сообщений: пользовательский текст оповещений теперь не ломается из-за символов Markdown.
